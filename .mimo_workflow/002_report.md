# Отчёт: Задание 002 — Hero Enhancement & Mobile Menu Fix

## Дата: 2025-06-11

## Задача 1: Мобильное меню

### Проблема
Бургер-иконка была статичной — не реагировала на клик, меню не открывалось.

### Решение
- Добавлен `useState` для управления состоянием меню
- При клике на бургер: три полоски анимируются в крестик (X) через CSS transitions
- Полноэкранный оверлей с glassmorphism-эффектом (backdrop-blur + тёмный фон)
- Содержит навигационные ссылки (Главная, Услуги, Контакты) с плавным скроллом
- Кнопка "Позвонить" с tel: ссылкой
- Telegram-контакт @DemonMSB
- Закрытие по клику на фон или по ссылке
- Блокировка скролла.body при открытом меню
- GSAP-анимации появления элементов меню (stagger)

## Задача 2: Обогащение Hero-секции

### Проблема
Первый экран был пустым — только текст, без визуального якоря.

### Решение

#### Заголовок
- Крупный H1: **"РЕМОНТ АВТОЭЛЕКТРИКИ"** с gradient clipping (glow-text)
- Два слова в две строки для максимального визуального веса
- Шрифт Space Grotesk, размер `clamp(32px, 12vw, 60px)` на мобильных

#### SVG-графика
- Интерактивная схема: три кольца (вращаются),电路 traces (анимация drawing), молнии (пульсация), floating dots (merцание)
- Две молнии: основная (большая, cyan gradient) и декоративные (меньшие, purple)
- Узлы на трассах с пульсирующей анимацией
- Весь SVG плавает вверх-вниз (GSAP sine.inOut)

#### CTA
- Главная кнопка: **"Вызвать электрика"** с телефонной иконкой, ссылка `tel:+79960314637`
- Вторичная: **"Все услуги"** outline-стиль

#### Layout
- Двухколоночная сетка: текст слева, графика справа
- На планшетах: графика перемещается над текстом
- На мобильных: всё в одну колонку, графика уменьшается
